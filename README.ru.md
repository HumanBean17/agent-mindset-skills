# Agent Mindset Skills

> 🇷🇺 Русский · 🇬🇧 [English](./README.md)

Набор **рабочих позиций** — каждая задаёт режим работы агента для конкретной задачи. Они отвечают на вопросы вроде: *Насколько автономно? Скорость или качество? Сделать самому или делегировать? Минимальные изменения или смелые ставки?*

Они **запускаются вручную** (например, `/surgeon` или названием позиции в вашем запросе) — вы выбираете позицию под ситуацию и переключаетесь, когда она меняется. Они написаны так, что *могут* также всплывать автоматически, когда задача явно подходит под одну из них, но предполагаемое использование — осознанный выбор. Если для задачи всплывает несколько, выберите одну — лучшее совпадение; одновременно должна действовать только одна позиция (поверх неё можно наслоить позицию-стиль вроде `surgeon` или `archivist`). Каждая позиция намеренно категорична: она оптимизирована под что-то одно и прямо говорит, чем сознательно жертвует. В этом компромиссе и весь смысл.

## Использование

Скопируйте каждый каталог `<позиция>/` в расположение навыков вашего агента, затем вызывайте по имени.

- **Claude Code:** скопируйте каталоги позиций в `~/.claude/skills/` (или в проектный `.claude/skills/`), затем выполните `/surgeon` (или назовите позицию в запросе).
- **Codex:** поместите под `~/.agents/skills/`.
- **Другие агенты:** это обычные файлы `SKILL.md` со стандартными полями `name` / `description` в frontmatter — кладите туда, где ваш агент ищет навыки.

Выбирайте одну позицию на задачу или наслаивайте позицию-стиль на исполняющую (см. [Комбинирование и противоречия](#комбинирование-и-противоречия)).

## Выбор под ситуацию

| Ситуация | Позиция |
|---|---|
| Нужен анализ или рекомендация — без изменений | `advisor` |
| Проверить на прочность план или изменения до того, как принять решение | `skeptic` |
| Одноразовый скрипт или жёсткий лимит времени | `sprinter` |
| Код библиотеки, публичный API или общая зависимость | `perfectionist` |
| Новый модуль / прототип, где ставка — на структуру | `trailblazer` |
| Хотфикс или хрупкий легаси-код — трогать как можно меньше | `surgeon` |
| Существующий код нужно перестроить, не меняя поведение | `refactorer` |
| Короткая, чётко очерченная задача — просто сделать, без лишних надстроек | `solo-operator` |
| Долгая сессия с распараллеливаемой работой — делегировать | `orchestrator` |
| Доверенная цель, от которой можно отойти, от начала и до конца | `autonomous-executor` |
| Задача неоднозначна — проясните замысел до начала работы | `clarifier` |
| Хотите оставаться в процессе, шаг за шагом | `pair-partner` |
| Долгий проект — нужно чисто передать контекст | `archivist` |

## Как они соотносятся

**Автономность** — кто ведёт каждый шаг:
`pair-partner` / `clarifier` (вы ведёте или сначала проясняете неоднозначность) → `autonomous-executor` (ведёт агент, вы отходите)

**Скорость ↔ Качество** — что вы оптимизируете:
`sprinter` (быстро, одноразово) ↔ `perfectionist` (всесторонне, отполированно)

**Делегирование** — делать vs. координировать:
`solo-operator` (делать всё самому) → `orchestrator` (делегировать сабагентам)

**Настрой** — конструктивный vs. антагонистический:
`advisor` (размышлять и рекомендовать) ↔ `skeptic` (атаковать и ломать)

**След (footprint)** — насколько масштабны изменения и на каком коде:

|                  | существующий код | новый (с нуля) |
|------------------|------------------|----------------|
| **трогать мало**     | `surgeon`        | `sprinter` ¹   |
| **перестроить**      | `refactorer`     | `trailblazer`  |

¹ Домашняя ось `sprinter` — скорость; он здесь, потому что одноразовый код минимален по структуре, а не потому что задача мала.

`archivist` — *стиль*-слой (сначала документация), наслаивается на любую из вышеописанных.

## Комбинирование и противоречия

Большинство позиций выбираются по одной на задачу, но позиции-стили наслаиваются чисто: выполните задачу `autonomous-executor` с дисциплиной `surgeon` или ведите заметки `archivist` во время `trailblazer`-спайка — назовите обе позиции в вызове.

Несколько пар — почти противоположности (`sprinter` ↔ `perfectionist`, `advisor` ↔ `skeptic`, `solo-operator` ↔ `orchestrator`, `surgeon` ↔ `refactorer`, `clarifier` ↔ `autonomous-executor`). Вызывать обе — противоречие; выберите ту, которую требует ситуация.
